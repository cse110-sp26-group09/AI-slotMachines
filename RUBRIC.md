# Evaluation Rubric

Each candidate is evaluated using the following criteria:

## 1. Functionality
- Does the app run in the browser?
- Do the spin mechanics work correctly?
- Are wins/losses displayed properly?
- Do major features behave as expected?

## 2. Simplicity / Correct Use of Tools
- Is the implementation clean and understandable?
- Does it use vanilla HTML, CSS, and JavaScript appropriately?
- Does the code avoid unnecessary complexity?

## 3. User Experience
- Is the interface clear and easy to use?
- Does the game feel engaging and enjoyable?
- Are feedback systems such as visuals, sound, and interaction satisfying?

## 4. Responsiveness / Bugs
- Does the app respond smoothly to user input?
- Are there broken features, visual glitches, or logic issues?
- Does the app remain usable without major errors?

## 5. Creativity / Theme Fit
- Does the slot machine successfully reflect the AI satire theme?
- Does it make “winning/spending tokens” feel playful and thematic?
- Does it stand out from other candidates in style or concept?

## 6. Polish
- Are the visuals cohesive?
- Are animations, sounds, and layout integrated well?
- Does the app feel complete rather than rough or unfinished?

## Overall Selection Priority
When choosing top candidates, we prioritize:
1. Functionality
2. User experience
3. Responsiveness / bug-free behavior
4. Polish
5. Creativity / theme fit
6. Simplicity
